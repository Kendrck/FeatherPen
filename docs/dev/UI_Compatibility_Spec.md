# FeatherPen 跨平台 UI 兼容性规范 V1.0
## 文档信息
版本：V1.0
用途：全平台UI渲染、功能降级、交互统一技术标准
适用载体：桌面端、Web端、移动端、小程序端
约束等级：项目强制UI基准规范，所有UI组件开发、迭代必须遵循本文档

## 1. 适用范围
覆盖项目全部UI渲染、进度展示、系统监控、多语言、数据导入导出、功能适配模块，适配全部官方支持平台与运行形态。

## 2. 核心设计原则
1. UI渲染层与底层业务逻辑完全解耦分离
2. 单套UI代码多端复用，禁止平台分支冗余代码
3. 环境不兼容时自动安全降级，不阻断主创作流程
4. 权限最小化、运行资源占用最小化
5. 全平台统一数据格式、统一交互行为标准

## 3. 支持平台与运行形态
### 兼容操作系统
Windows10/11、macOS、全系列Linux发行版、国产Linux、Android、iOS
### 运行分发形态
桌面EXE便携包、Linux AppImage/deb/rpm、macOS DMG、浏览器扩展、Web网页、Android APK、iOS IPA、微信小程序、QQ小程序
### 扩展文件支持
VSCode扩展、VSIX插件、私有.ftpen工程扩展、json/langpack语言扩展包

## 4. UI定时刷新统一规范
### AI写作进度刷新项
默认刷新间隔：1秒
参数合法区间：1 ~ 999秒
非法数值自动回滚默认值1秒
### 系统硬件监控刷新项
默认刷新间隔：5秒
参数合法区间：1 ~ 999秒
非法数值自动回滚默认值5秒

## 5. 数据导入导出统一规范
### 支持导入格式
txt、md、json、外置语言包、全局配置文件、书籍工程备份压缩包
### 支持导出格式
txt、md、json、zip工程备份、系统配置文件
### 强制统一规则
全平台统一UTF-8文本编码；原生支持中文路径；文件读写异常安全终止，不篡改、不损坏用户本地书籍数据。

## 6. 多语言适配规范
内置4套基础语言包，支持外置自定义langpack语言扩展；跨平台加载逻辑统一；语言包文件损坏、缺失时自动降级回中文简体。

## 7. 小程序端专属兼容规则
### 环境原生限制
无法读取CPU、内存、显卡等系统硬件信息；不支持本地插件、第三方扩展加载
### 自动适配降级策略
1. 全局隐藏系统硬件监控面板
2. 完整保留小说核心创作、编辑功能
3. 保留进度实时刷新、语言切换、工程导入导出
4. 隐藏扩展管理功能入口
5. 使用小程序本地存储托管用户个性化配置

## 8. 全平台统一功能降级策略
### 桌面端（Windows/macOS/Linux）
解锁全部UI功能；硬件指标读取失败时自动隐藏对应监控模块，不弹窗报错。
### Web网页端
无系统硬件读取权限，全局隐藏硬件监控面板；完整保留小说创作、参数配置、导出功能。
### 移动端（Android/iOS）
自适应自适应缩放布局；关闭高阶监控附加功能，保障基础小说创作流程完整可用。
### 小程序端
极简安全运行模式，仅保留章节创作、文本编辑、内容导出、语言切换基础能力。

## 9. 通用异常自动降级规则
1. 硬件信息采集失败 → 隐藏对应监控展示控件
2. 当前运行环境不支持扩展组件 → 隐藏扩展管理入口
3. 用户输入刷新间隔参数非法 → 重置为规范默认值
4. 全局配置文件损坏丢失 → 自动加载config.yaml内置默认配置
5. 外置语言包加载异常 → 强制回退中文简体
6. 导入工程文件格式异常/损坏 → 安全拦截，提示用户校验文件

## 10. Linux系统深度兼容标准
1. 统一配置存储目录：`~/.config/FeatherPen/`
2. 无强制依赖特殊系统字体、第三方系统库
3. 原生适配高分屏、多显示器多窗口场景
4. 兼容X11、Wayland双显示服务协议

## 11. UI性能硬性约束标准
1. 常规UI空闲渲染CPU占用 ≤ 2%
2. 单UI组件内存占用上限 ≤ 128MB
3. 全局定时器生命周期绑定父窗口，窗口销毁同步销毁定时器，杜绝内存泄漏
4. 小程序端整体内存占用上限 ≤ 80MB

## 12. FloatMonitorWidget 悬浮监控小窗口组件规范
### 12.1 文件归属
新增组件源码路径：`ui/float_widget.py`
组件定位：独立顶层无边框窗口组件，无小说工作台页面依赖，拥有独立生命周期。

### 12.2 主程序集成规范
1. 在 `ui/main_window.py` 工具菜单栏新增悬浮监控开关入口；
2. 采用单例模式全局管理悬浮窗口实例，同一时间仅允许存在一个悬浮窗口对象，规避多实例资源冲突。

### 12.3 跨平台适配规则
| 运行平台 | 组件启用逻辑 |
|--------|------------|
| Windows / Linux / macOS | 完整启用全部功能，支持置顶、拖拽移动、实时数据刷新 |
| Android 移动端 | 不实例化组件，自动隐藏悬浮窗口入口 |
| Web网页端 | 正常加载组件，永久禁用置顶窗口属性，仅保留监控数据展示 |
| 小程序端 | 完全移除悬浮窗口菜单入口，不加载组件代码 |

### 12.4 模块通信标准
悬浮窗口与主窗口统一使用Qt信号槽完成交互逻辑；禁止跨类直接调用对象方法，实现业务代码完全解耦。

### 12.5 资源性能约束
1. 闲置状态下定时器仅周期轮询内存数据，无高频磁盘IO、网络请求；
2. 组件常驻内存占用上限 10MB，不占用模型推理、章节生成硬件资源，不干扰主线创作流程。

## 13. 文档约束力条款
本文档为项目最高UI兼容强制标准，所有UI组件迭代、新增、修改必须严格遵循规范；禁止私自修改平台降级逻辑、定时刷新规则、数据交互格式标准。
