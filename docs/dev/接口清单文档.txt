一、文档说明
================================================================================

本文档定义 FeatherPen 羽笔 V1.0.0 所有后端 API 接口的完整规范，包括请求
格式、响应格式、错误码、认证方式等。所有接口遵循 RESTful 风格，统一使用
JSON 数据格式。


================================================================================
二、通用规范
================================================================================

2.1 接口前缀
-------------------------------------------------------------------------------
所有业务接口固定前缀：/api/v1/
禁止自定义前缀。


2.2 通信协议
-------------------------------------------------------------------------------
┌─────────────────┬──────────────────┬────────────────────────────────────────┐
│ 类型            │ 协议             │ 说明                                   │
├─────────────────┼──────────────────┼────────────────────────────────────────┤
│ 普通请求        │ HTTP POST/GET    │ 同步请求，立即返回结果                 │
│ 流式生成        │ SSE              │ 低延迟流式输出，实时返回生成片段       │
└─────────────────┴──────────────────┴────────────────────────────────────────┘


2.3 数据格式
-------------------------------------------------------------------------------
- 所有请求和响应：JSON 格式
- 字符编码：UTF-8
- 时间格式：ISO 8601（如 2026-07-20T14:30:00Z）


2.4 错误处理规范
-------------------------------------------------------------------------------
标准HTTP状态码：
  200  = 请求成功
  400  = 请求参数错误
  401  = 未认证
  403  = 权限不足
  404  = 资源不存在
  500  = 服务器内部错误

自定义业务错误码：
  4001  = Model Not Loaded        推理引擎未加载模型或模型文件损坏
  4002  = Invalid Context ID      上下文ID不存在或已过期失效
  4003  = Insufficient Points     积分余额不足，无法完成操作
  4004  = Daily Limit Exceeded    当日积分消耗已达上限
  4005  = Permission Denied       当前会员等级无此操作权限
  5001  = GPU OOM                 GPU显存溢出
  5002  = Engine Timeout          推理引擎响应超时
  5003  = Engine Busy             生成任务队列已满，请求排队重试

统一异常返回结构：
  {
    "code": 4001,
    "message": "Model Not Loaded",
    "detail": "推理引擎未加载模型或模型文件损坏",
    "timestamp": "2026-07-20T14:30:00Z"
  }


2.5 认证方式
-------------------------------------------------------------------------------
┌─────────────────┬──────────────────┬────────────────────────────────────────┐
│ 模式            │ 认证方式         │ 说明                                   │
├─────────────────┼──────────────────┼────────────────────────────────────────┤
│ 本地离线        │ 无认证           │ Lv0 无名，绑定主板ID                   │
│ 登录会员        │ Session/Token    │ 登录后获取会话标识                     │
└─────────────────┴──────────────────┴────────────────────────────────────────┘

三、核心接口清单
================================================================================

3.1 触发AI文本流式生成
-------------------------------------------------------------------------------
接口路径：POST /api/v1/generate
功能描述：触发AI文本流式生成任务，实时返回生成内容

请求头：
  Content-Type  = application/json  （必填）
  X-Session-ID  = 当前会话ID         （必填）

请求体：
  {
    "prompt": "请继续撰写以下故事...",
    "context_id": "ctx_20260720_001",
    "stream": true,
    "temperature": 0.7,
    "max_tokens": 4000
  }

参数说明：
  ┌─────────────────┬──────────┬──────────┬────────────────────────────────┐
  │ 参数名          │ 类型     │ 必填     │ 说明                           │
  ├─────────────────┼──────────┼──────────┼────────────────────────────────┤
  │ prompt          │ string   │ 是       │ 生成提示词，建议包含前文摘要   │
  │ context_id      │ string   │ 是       │ 上下文唯一ID，用于跟踪会话     │
  │ stream          │ boolean  │ 是       │ 是否启用流式输出，默认true     │
  │ temperature     │ float    │ 否       │ 生成温度，范围0.0-1.0，默认0.7 │
  │ max_tokens      │ int      │ 否       │ 最大生成token数，默认4000      │
  └─────────────────┴──────────┴──────────┴────────────────────────────────┘

响应格式：SSE 流式事件

SSE数据包结构：
  {
    "chunk": "生成的文本片段...",
    "status": "processing",
    "token_usage": 42
  }

字段说明：
  chunk（string）      ：当前生成的文本片段
  status（string）     ：状态：processing / complete / error
  token_usage（int）   ：当前已消耗的Token数（仅processing状态返回）

结束标识：
  {
    "chunk": "",
    "status": "complete",
    "total_tokens": 1523
  }

错误响应（SSE格式）：
  {
    "error": "GPU显存溢出，请降低生成参数",
    "status": "error",
    "code": 5001
  }

调用示例（JavaScript）：
  const client = new FeatherPenClient();
  await client.generate(
    "请为以下章节生成大纲...",
    "ctx_001",
    (chunk) => { console.log(chunk); }
  );


3.2 获取引擎实时运行状态
-------------------------------------------------------------------------------
接口路径：GET /api/v1/status
功能描述：获取推理引擎的实时运行状态，用于监控仪表盘

请求头：
  X-Session-ID = 当前会话ID（必填）

请求参数：无

响应示例：
  {
    "code": 0,
    "message": "success",
    "data": {
      "gpu_usage": 45.2,
      "gpu_memory_mb": 4096,
      "memory_mb": 2048,
      "cpu_percent": 32.5,
      "model_loaded": true,
      "model_name": "qwen2.5-14b-instruct-1m",
      "uptime_seconds": 3600,
      "queue_size": 0
    }
  }

字段说明：
  ┌─────────────────┬──────────┬────────────────────────────────────────────┐
  │ 字段名          │ 类型     │ 说明                                       │
  ├─────────────────┼──────────┼────────────────────────────────────────────┤
  │ gpu_usage       │ float    │ GPU利用率百分比，无GPU返回-1               │
  │ gpu_memory_mb   │ int      │ GPU显存占用（MB）                          │
  │ memory_mb       │ int      │ 系统内存占用（MB）                         │
  │ cpu_percent     │ float    │ CPU利用率百分比                            │
  │ model_loaded    │ boolean  │ 模型是否已加载就绪                         │
  │ model_name      │ string   │ 当前加载的模型名称                         │
  │ uptime_seconds  │ int      │ 引擎运行时长（秒）                         │
  │ queue_size      │ int      │ 等待队列中的任务数                         │
  └─────────────────┴──────────┴────────────────────────────────────────────┘


3.3 多端项目数据同步
-------------------------------------------------------------------------------
接口路径：POST /api/v1/sync
功能描述：多端（桌面、Web、移动端）项目大纲、设定、进度数据同步

请求体：
  {
    "project_id": "book_20260720_001",
    "sync_type": "full",
    "data": {
      "outline_version": 3,
      "last_chapter": 15,
      "total_sections": 450,
      "characters": [
        { "name": "主角A", "role": "protagonist" },
        { "name": "配角B", "role": "supporting" }
      ],
      "timeline": [
        { "event": "故事开始", "chapter": 1 },
        { "event": "转折点", "chapter": 15 }
      ]
    }
  }

参数说明：
  ┌─────────────────┬──────────┬──────────┬────────────────────────────────┐
  │ 参数名          │ 类型     │ 必填     │ 说明                           │
  ├─────────────────┼──────────┼──────────┼────────────────────────────────┤
  │ project_id      │ string   │ 是       │ 工程唯一ID                     │
  │ sync_type       │ string   │ 否       │ 同步类型：full / incremental   │
  │ data            │ object   │ 是       │ 同步数据内容                   │
  └─────────────────┴──────────┴──────────┴────────────────────────────────┘

响应示例：
  {
    "code": 0,
    "message": "同步成功",
    "data": {
      "sync_time": "2026-07-20T14:30:00Z",
      "version": 4,
      "conflicts": []
    }
  }

字段说明：
  sync_time（string） ：服务端同步时间
  version（int）      ：新的数据版本号
  conflicts（array）  ：冲突列表（若有）


3.4 获取会员信息
-------------------------------------------------------------------------------
接口路径：GET /api/v1/member/info
功能描述：获取当前登录用户的会员等级、积分、权限信息

请求头：
  X-Session-ID = 当前会话ID（必填）

响应示例：
  {
    "code": 0,
    "message": "success",
    "data": {
      "uid": "88888888",
      "level": 8,
      "level_name": "Lv8 传奇",
      "points": 999999999,
      "daily_consumed": 5,
      "daily_limit": -1,
      "permissions": {
        "world_check": true,
        "monitor_full_data": true,
        "monitor_token_detail": true,
        "monitor_ai_progress": true
      },
      "limits": {
        "max_send_chapter": 60,
        "max_send_outline": 100,
        "max_output_chapter": 1000
      }
    }
  }


3.5 积分扣费
-------------------------------------------------------------------------------
接口路径：POST /api/v1/points/consume
功能描述：执行积分扣费操作，用于生成、校验等功能

请求体：
  {
    "uid": "88888888",
    "amount": 2,
    "operation_type": "gen_chapter",
    "description": "生成第3章第2节正文"
  }

参数说明：
  ┌─────────────────┬──────────┬──────────┬────────────────────────────────┐
  │ 参数名          │ 类型     │ 必填     │ 说明                           │
  ├─────────────────┼──────────┼──────────┼────────────────────────────────┤
  │ uid             │ string   │ 是       │ 用户ID                         │
  │ amount          │ int      │ 是       │ 扣费积分数                     │
  │ operation_type  │ string   │ 是       │ 操作类型枚举（见下表）         │
  │ description     │ string   │ 否       │ 操作描述                       │
  └─────────────────┴──────────┴──────────┴────────────────────────────────┘

operation_type 枚举值：
  ┌─────────────────┬──────────────────────────────────┬────────────────┐
  │ 值              │ 说明                             │ 标准积分       │
  ├─────────────────┼──────────────────────────────────┼────────────────┤
  │ gen_chapter     │ 单节正文生成                     │ 2              │
  │ sort_role       │ 角色整理归档                     │ 1              │
  │ gen_timeline    │ 时间线生成/修正                  │ 3              │
  │ full_check      │ 手动全局全书校验                 │ 10             │
  │ auto_check      │ 自动触发全书校验                 │ 5              │
  └─────────────────┴──────────────────────────────────┴────────────────┘

响应示例：
  {
    "code": 0,
    "message": "扣费成功",
    "data": {
      "before_balance": 100,
      "after_balance": 98,
      "daily_consumed": 7,
      "daily_limit": 10
    }
  }

错误响应：
  {
    "code": 4003,
    "message": "Insufficient Points",
    "detail": "当前积分余额不足，需要2积分，当前余额0",
    "timestamp": "2026-07-20T14:30:00Z"
  }
