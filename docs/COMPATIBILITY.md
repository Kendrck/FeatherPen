# FeatherPen 国标开发贡献规范（GB/T 8567-2006）
## 一、代码编写规范
1. 所有代码遵循PEP8 Python编码标准 + 项目国标强制规则
2. 代码无冗余、无调试残留、无废弃逻辑、无临时注释
3. 所有核心逻辑配套标准化业务注释，可追溯、可解读
4. 严格分层解耦，禁止跨层级逻辑混杂

## 二、模块开发规范
1. 单次仅开发单个三级模块，完成后验收归档再迭代下一个模块
2. 所有新增逻辑必须同步更新docs对应归档文档
3. 所有功能必须100%对齐FeatherPen V1.0.0国标开发文档
4. 禁止简化、删减、篡改任何预设业务规则

## 三、交付验收规范
1. 代码纯净：无迭代记录、无调试信息、无冗余空行
2. 文档同步：代码与归档文档完全一致
3. 功能合规：完全匹配国标需求，无逻辑偏差
4. 可直接运行：无语法错误、无依赖缺失、无逻辑漏洞

## 四、禁止行为
1. 禁止修改项目一、二级固定目录架构
2. 禁止留存任何临时测试代码与调试注释
3. 禁止不同步文档交付代码
4. 禁止自由拓展非国标业务逻辑
