# FeatherPen 羽笔 V1.0.0 国标强制开发规范
## 文档基准优先级（严格不可颠倒）
1. 《GB/T 8567-2006 计算机软件文档编制规范》【全局最高强制国标】
2. FeatherPen 羽笔 V1.0.0 国标完整版开发文档【业务唯一需求基准】
3. 本 PROJECT_RULES.md 全局强制编码约束【开发执行唯一准则】
4. 仓库原生一、二级固定目录架构【顶层架构永久锁定】

## 一、目录架构国标合规总则
### 1. 层级锁定铁律
- 一级目录、二级目录永久锁定，禁止任何新增、删除、重命名、移动、合并操作
- 仅开放三级及以下目录合规新建/修改权限，所有功能拓展必须在现有二级目录下三级层级实现
- 所有三级新增文件/子目录必须贴合业务场景、配套标准化注释、同步docs归档文档，无文档视为无效交付

### 2. 合规判定标准
当前项目原生一、二级目录完全符合GB/T 8567商用软件工程规范，架构定型、无冗余、无错乱，可永久作为项目基准架构，仅支持三级合规拓展。

## 二、代码交付硬性强制规则
### 1. 路径输出规范
- 所有文件路径必须完整书写：FeatherPen/根路径/二级目录/三级目录/文件名.后缀
- 禁止简写路径、相对路径、省略层级、自定义路径
- 禁止修改一、二级目录结构，仅允许三级文件迭代优化与合规新增

### 2. 纯净代码输出规范（强制执行）
所有代码交付必须彻底清理以下冗余内容，零残留：
- 删除所有迭代日志、版本修正记录、bug修复备注、调试过程说明
- 删除所有临时测试代码、注释掉的废弃代码、调试print语句
- 删除所有TODO、草稿备注、临时优化方案、未验证逻辑说明
- 删除AI生成过程解释、代码修改说明、冗余空行、重复导入
- 最终代码仅保留：标准业务源码 + 国标规范化业务注释

### 3. 代码注释统一标准
- 文件头部：必须标注文件功能、所属业务模块、对应国标文档章节
- 所有类、核心函数、关键逻辑行：逐条添加业务注释，说明作用、适配规则、参数含义
- 配置文件：关键参数行内注释，标注国标依据、默认规则、业务约束
- 注释仅描述最终业务功能，禁止留存调试、迭代、临时相关内容

### 4. 代码结构规范
- 严格分层解耦：UI层仅渲染、Src层承载核心业务、Database层仅数据读写
- 禁止业务逻辑侵入UI层、禁止数据操作混入核心生成逻辑
- 所有权限校验、剧情校正、积分扣费、数值拦截逻辑禁止简化、删减分支
- 所有变量、函数、文件命名遵循Python标准化PEP8规范+项目国标命名规则

## 三、文档归档强制铁律
### 1. 同步规则
任何代码新增、修改、优化、三级文件新建，必须同步更新docs对应归档文档，无文档同步则代码交付无效，强制重写。

### 2. Docs文档固定职责
- README.md：项目基础介绍、快速上手、整体架构概述
- YESAPI_ACCOUNT.md：账号权限、设备登录、白名单体系规范
- STRUCTURE.md：项目目录架构、世界树五级层级规范
- DEVELOP.md：环境搭建、运行流程、单元测试规范
- API.md：全局所有接口唯一归档文档
- COMPATIBILITY.md：全平台适配、跨系统兼容规范
- PRESS_TEST.md：Lv9压测模式、极限解禁、亿级测试规范
- WORLD_TREE.md：五级世界树架构、校正、记忆、插叙核心规范
- TEMPLATE.md：全局配置参数释义、国标约束模板
- CONTRIBUTING.md：代码提交、格式化、开发协作规范

### 3. 文档规范
所有docs文档严格遵循GB/T 8567标准，结构分层清晰、参数表格化、逻辑结构化，无草稿、无临时内容、无未验证猜想。

## 四、业务代码强制规范（不可篡改）
1. 四层校验永久保留：UI拦截 → 参数校验 → 后端业务校验 → 数据库校验
2. 世界树五级层级锁定规则、100节校正机制、三层冷热记忆机制完整落地，禁止简化
3. 普通账号5节上限、Lv9压测解禁、积分扣费豁免逻辑完整分支保留
4. SQLite仅存元数据、小节正文外置TXT存储的亿级架构永久生效
5. 本地API自动清洗、非法参数回滚、容错逻辑永久保留

1. 禁止修改、新增、删除项目一、二级目录
2. 禁止保留任何迭代、调试、临时、废弃代码及注释
3. 禁止简化、删减、篡改国标规定的业务逻辑
4. 禁止代码交付不同步归档文档
5. 禁止自由新增无效文件、冗余模块、非国标逻辑
